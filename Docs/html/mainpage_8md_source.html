6 ##Overview##
7 This package provides methods and classes for analysing the output of mitgcm simulations.

11 ##To install##
12 Download and install python. I'd recommend [Annaconda](https://store.continuum.io/cshop/anaconda/), it's a nice self-contained python distribution.
13 
14 
15 The code can be installed through conda using
16 
17 \code
18 conda install -c https://conda.anaconda.org/edoddridge mitgcm
19 \endcode
20 
21 Or, it can be installed by:
22 
23 Downloading the code (either by downloading the repo or cloning it into a directory)
24 
25 Navigating to the download directory and running:
26 
27 \code
28 python setup.py install
29 \endcode
30 
31 in the terminal

34 ###To install on ARCHER###
35 The way that python is setup on ARCHER makes this a little trickier.
36 
37 Firstly, make sure you've loaded the Anaconda module
38 
39 \code
40 module load anaconda
41 \endcode
42 
43 Then, create a conda environment called ‘VENV’ with base python installed inside it
44 
45 \code
46 conda create --name VENV python
47 \endcode
48 
49 activate this environment with
50 
51 \code
52 source activate VENV
53 \endcode
54 
55 remove the PYTHONPATH and PYTHONHOME variables
56 
57 \code
58 unset PYTHONPATH
59 unset PYTHONHOME
60 
61 conda install -c https://conda.anaconda.org/edoddridge mitgcm
62 \endcode
63 
64 If you want to use ipython notebooks, you’ll also need to run
65 
66 \code
67 conda install jupyter
68 \endcode
69 
70 
71 It's also worth adding
72 \code
73 source activate VENV
74 unset PYTHONPATH
75 unset PYTHONHOME
76 \endcode
77 
78 to your .bashrc file, so that you don't have to manually do this everytime you log in.

89 ###To use notebooks on ARCHER###
90 By using port forwarding when logging in to ARCHER it is possible to run a notebook on ARCHER and access it locally. It's best to run these on the serial nodes. To do this ssh into the login nodes
91 
92 \code
93 ssh -X -L11111:localhost:11111 USERNAME@login.archer.ac.uk
94 \endcode
95 
96 and from there, ssh into the serial nodes
97 
98 \code
99 ssh -L11111:localhost:11111 espp1
100 \endcode
101 
102 and run
103 \code
104 ipython notebook --no-browser --port 11111
105 \endcode
106 
107 to launch the notebook server. Then point your web browser towards "http://localhost:11111/" to use the notebook.
